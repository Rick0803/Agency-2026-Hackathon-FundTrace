# 2. Ghost Capacity

## Problem Definition

Which funded organizations show no evidence of actually being able to deliver what they were funded to do?

These are **not "zombies"** (which eventually fail) — they are **ghost-capacity entities**: organizations that persist indefinitely, continue receiving public funding, and produce no measurable output.

Observed characteristics:

- No employees
- No physical presence
- No revenue beyond government transfers
- Expenditures flow almost entirely to compensation for a small number of individuals, or to transfers to other entities

---

## How the Problem Will Be Tackled

The core challenge is that ghost capacity has no single definitive signal — it is a *combination* of financial, operational, and temporal patterns that together paint a picture of dysfunction. The approach layers three complementary methods:

**1. Rule-based profiling** — Hard signals that are individually suspicious: zero employees, near-zero program spend, >90% revenue from government transfers. These produce explainable flags a human investigator can immediately verify.

**2. Unsupervised anomaly detection** — Isolation Forest trained on a peer cohort of government-funded charities, so "anomalous" means unusual *among similar organizations*, not just unusual in absolute terms. This surfaces non-obvious patterns that no predefined rule would catch.

**3. Composite Ghost Score** — A weighted combination of all dimensions into a single 0–1 score, decomposed by feature contribution so each flagged entity comes with a clear, citable rationale. Outputs are audit-ready.

Detection is dual-mode:
- **Targeted investigation** — natural language querying ("show entities with no employees and over $500K federal funding") for human-led exploration
- **Broad scan** — unsupervised sweep across the full entity universe to surface non-obvious cases

Persistence is treated as a first-class signal throughout: a pattern holding across 3–5 consecutive fiscal years is categorically more significant than a single-year anomaly. Change-point detection (ruptures) identifies the specific year a ghost pattern began, which is as important as the pattern itself.

## Implemented Plan

### Tech Stack

| Layer | Tool |
|---|---|
| Database | PostgreSQL (CRA + FED schemas) |
| Data retrieval | Python + psycopg2, pandas |
| Anomaly detection | PyOD — Isolation Forest (primary), HBOS / LOF (benchmarks) |
| Change-point detection | ruptures (Pelt / Dynp, L2 model) |
| Explainability | Feature-level score decomposition (weighted dimension contributions) |
| LLM orchestration | Claude (Anthropic SDK), agentic tool-use loop |
| Natural language interface | LLM-interpreted query → allowlisted SQL (no raw LLM SQL) |
| Reporting | RAG over government report templates + CRA program descriptions |
| Frontend | Streamlit |

### High-Level Steps

1. **Data ingestion** — Pull CRA T3010 filings (financials, compensation, qualified donees) and federal grants/contributions into PostgreSQL

2. **Feature engineering** — Compute per-entity, per-fiscal-year ratios:
   - Government revenue dependency (gov transfers / total revenue)
   - Program delivery ratio (program spend / total expenses)
   - Compensation burden (compensation / total expenses)
   - Pass-through transfer ratio (transfers out / total expenses)
   - Employee count
   - Amendment creep (grant value growth via amendments vs new competitive awards)

3. **Ghost Score computation** — Weighted composite of all dimensions (0–1), averaged across available fiscal years to capture persistence; change-point detection to identify regime-change year

4. **Unsupervised anomaly detection** — Isolation Forest trained on peer cohort (same entity type, same province); normalised ECOD-style outlier score supplements the rule-based Ghost Score

5. **Sector benchmarking** — Score interpreted relative to peer cohort, not absolute thresholds; a 70% gov dependency ratio in one sector may be normal and anomalous in another

6. **Explainability generation** — Per-dimension contribution breakdown; each flag is backed by specific cited figures (e.g., "97% gov dependency for 4 consecutive years")

7. **Analysis layer** — Classify entities by severity (LOW / MEDIUM / HIGH / CRITICAL); cluster similar ghost patterns; compute systemic signals:
   - Funding department exposure (which departments are most at risk)
   - Geographic clustering (province / city concentration)
   - "Money at risk" aggregate (total public dollars to high-score entities)

8. **Reporting** — Two output levels:
   - *Entity report* — detailed breakdown per organization: anomalies, supporting evidence, feature contributions, historical behavior, RAG-grounded narrative using government report template
   - *Portfolio report* — total flagged entities, severity distribution, estimated financial exposure, department and geographic breakdowns

## Questions It Answers

**Entity-level**
- Is this organization a ghost-capacity entity? How severe?
- Which specific signals triggered the flag — and with what supporting evidence?
- When did the ghost pattern begin?
- Is it getting persistently re-funded through amendment creep rather than competitive grants?
- How anomalous is this entity compared to similar government-funded organizations?

**Portfolio-level**
- How much public money is currently flowing to high-risk ghost-capacity entities?
- Which federal departments have the greatest exposure?
- Are ghost-capacity entities concentrated in specific provinces, cities, or sectors?
- Is this a diffuse national pattern or localized to specific funding streams?
- What is the estimated financial exposure at each severity tier?

**Investigative**
- Which entities should be prioritized for audit or closer review?
- What is the complete evidentiary basis for flagging a specific organization?
